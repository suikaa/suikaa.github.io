git常用命令总结

2021-03-29 工具 GitHub, git 0 字数统计: 1.7k(字) 阅读时长: 6(分)

一、新建仓库

以GitHub作为远程仓库为例.

git init

在新建项目的目录下执行git init，即可将该目录初始化为一个git仓库。

git remote

关联远程仓库，首先需要在设置中添加个人的SSH key。
Mac的SSH key生成方式如下，详细配置方法见GitHub帮助文档。

完成SSH key设置后，通过如下命名实现本地与远程仓库的关联。

git clone

远程仓库已创建的情况下，可以从远程仓库拷贝代码到本地，此时会自动创建本地与远程的关联。

二、将本地变更内容推送到远程仓库

变更推送需要经历四个过程：

  1. 保存工作区（项目在本地的文件夹）的变更
  2. 将变更提交到暂存区 —— git add
  3. 将暂存区的所有内容提交到git库生成版本号 —— git commit
  4. 将本地分支的变更提交到远程仓库的对应分支 —— git push

git status

查看文件状态

git add

提交之后文件状态由unstage变为staged。

git commit

建议采用最小提交原则，比如有两个bug需要修复，那么分别执行两次git commit，如果有两个功能点也一样，这样方便回溯。当然必须确保每次commit的功能点是完备的，代码可以正常运行，不要commi只完成了一半的工作。

git commit后会生成一个版本号。

git push

至此实现将本地变更推送到远程仓库。

三、回退操作

在实际工作中，因为误操作等原因，我们需要回退已提交的内容，根据文件所处的不同阶段可以采用下列操作。

git restore

git clean

快捷删除新增文件（untracked files）

git reset

如果变更已经commit，甚至push到远程仓库，此时要用到git reset操作执行版本回退。

git revert

如果存在1、2、3这三个版本，需要撤销版本2并保留版本3的提交，可以用git revert。

此时会生成新的版本4提交，而不会把版本2的提交删除。

git merge –abort

撤销合并分支操作。

四、分支管理

实际开发中，我们一般不会直接在master分支上提交修改，而是根据产品规划版本创建版本的开发分支。待分支开发完成，测试通过后，上线前再合并到master。
因此涉及分支的创建、切换、合并。

git pull

对于多人协作项目，该命令主要有两种使用场景：

  1. 初次同步远程分支的内容。
  2. 在每次修改代码前，同步远程分支的内容，以免本地提交时与远程文件发生冲突。

git branch

git checkout

git merge

假设我们需要把dev分支的内容合并到master，需要确保当前工作区在master，再执行git merge dev。此时dev分支commit记录也会合并到master。

git merge –abort可撤销合并操作

git stash

在开发中可能遇到这样的场景：正在dev分支上进行开发，但是发现了一个bug需要紧急修复上线，而dev分支上的内容还无法commit，这时就可以用到git stash的暂存功能。
