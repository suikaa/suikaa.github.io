ECMAScript中的object（1）广义的对象

2021-04-02 JavaScript ECMAScript 0 字数统计: 1.5k(字) 阅读时长: 5(分)

1. 面向对象的语言

ECMAScript是基于对象的：基础语言与环境都由对象提供，ECMAScript程序就是交流对象的集群。在ECMAScript中，对象是零或多个属性的集合（set），属性是其他对象、原始值、以及函数的容器。每个属性都有一些特性来决定属性的用法。例如，当一个的属性的Writable特性设置为false时，任何试图给这个属性赋值的操作都将失败。

2. 引用类型：Object

ECMAScript2020定义的语言类型有八种，其中七种为原始类型（Primitive Type），又称基本类型，包括：Undefined, Null, Boolean, String, Symbol, Number, BigInt。除了原始类型，其他都属于引用类型（Reference Type）：Object。

2.1. 对象的属性

上文提到，对象是属性的集合。属性由名称(key)、值（value）和一组特性组成。

  • key可以是String或Symbol类型。
  • 有一种特殊的类数组对象的key是整数Number类型，只能用方括号访问。
  • 可以用Object内置方法Object.getOwnPropertyDescriptor(O, P)查看特性值。

2.2. 数据属性和存取器属性

属性又分为数据属性和存取器属性。

  • 数据属性(data property)的值可以是ECMAScript任意语言类型的值，有4个特性：Value、Writable、Enumerable、Configurable。
  • 存取器属性(accessor property)的值是两个方法：getter和setter，有4个特性：Get、Set、Enumerable、Configurable。

3. 普通对象和怪异对象

3.1. 普通对象

  1. 所有的普通对象都有指定的内部方法（见ECMAScript2020-6.1.7.2 table 6）。这些内部方法不属于ECMAScript语言本身，而是用来描述语言的内部执行逻辑，在ECMAScript中用双中括号[[]]来表示。

  2. 拥有[[Call]]内部方法的对象称为函数对象（function object）。函数用函数名+小括号来调用，如functionName()。

  3. 拥有[[Construct]]内部方法的对象称为构造器（constructor）。支持[[Construct]]的对象也必然支持[[Call]]，意味着构造器必然是一个函数对象，因此构造器又称构造函数。构造器都有一个叫做prototype的属性，其中定义了可以被其实例继承的属性和方法。用new操作符可以创建构造器的实例对象，如new Date()创建了一个新的Date对象。是否可以像函数那样调用构造器——即不加new操作符——取决于构造器本身，比如Date()会返回当前日期和时间的字符串表达式，而非对象。

3.2. 怪异对象

除了上述普通对象，其他都是怪异对象（exotic object）：

  • Bound Function Exotice Objects
  • Array Exotic Objects
  • String Exotic Objects
  • Arguments Exotic Objects
  • Integer-Indexed Exotic Objects
  • Module Namespace Exotic Objects
  • Immutable Prototype Exotic Objects

4. 内置对象&全局对象

内置对象（built-in objects）又称标准库，一旦ECMAScript脚本或模块开始执行，它们就可以直接使用。其中，全局对象是执行程序全局环境的一部分，其他的内置对象要么是全局对象的初始属性，要么是内置对象的属性。

全局对象（global object）具有以下特征：

  • 在进入执行上下文之前已经被创建
  • 没有[[Construct]]内部方法，不能用new操作符作为构造器使用
  • 没有[[Call]]内部方法，不能作为函数调用
  • 具有[[Prototype]]内部slot，其值由宿主环境决定（host-defined）
  • 除了本规范定义的属性外，可能还有宿主环境定义的属性，可能包括属性值是全局对象本身的属性。
